# 入れ子構造

## このステップのゴール

- ブロック要素とインライン要素を使い分けられます
- 有効な入れ子ルールに従ってHTMLを書けます
- void要素を正しく使えます

## 前提知識・環境

- 02-text を完了していること

## 実践

`index.html` にコードを書きながら進めます。変更はLive Serverで確認します。

### 1. ブロック要素とインライン要素の違いを確認する

HTML要素には大きく2種類あります。次のコードをブラウザで確認します。

```html
<p>これはブロック要素（p）です。前後に改行が入ります。</p>
<p>2つ目の段落です。</p>

<span>これはインライン要素（span）です。</span>
<span>横に並びます。</span>
```

| 種別 | 特徴 | 代表的な要素 |
| --- | --- | --- |
| ブロック要素 | 前後に改行が入り、横幅いっぱいに広がる | `div`・`p`・`h1`〜`h6`・`ul`・`li` |
| インライン要素 | 前後に改行が入らず、コンテンツの幅になる | `span`・`a`・`strong`・`em`・`img` |

### 2. 入れ子ルールを守る

ブロック要素はインライン要素を内包できますが、逆はできません。また `p` の中にブロック要素は入れられません。

次のコードは無効な入れ子です。ブラウザが自動的に修正しますが、意図しない構造になります。

```html
<!-- 無効: p の中に div は入れられない -->
<p>
  <div>この入れ子は無効です</div>
</p>

<!-- 無効: span の中に p は入れられない -->
<span>
  <p>この入れ子も無効です</p>
</span>
```

ブラウザの開発者ツールでElementsタブを開き、実際のDOM構造を確認します。記述した構造と異なる形にブラウザが修正していることがわかります。

正しい入れ子の例です。

```html
<!-- 有効: ブロック要素の中にインライン要素 -->
<p><strong>強調テキスト</strong>を含む段落です。</p>

<!-- 有効: ブロック要素の中にブロック要素 -->
<div>
  <p>divの中の段落です。</p>
</div>
```

### 3. void要素を使う

**void要素**は終了タグを持たない要素です。子要素やテキストを持ちません。

```html
<!-- 水平線 -->
<hr />

<!-- 改行 -->
<p>1行目<br />2行目</p>

<!-- 画像（後のステップで詳しく学ぶ） -->
<img src="image.jpg" alt="説明文" />

<!-- フォームの入力欄（後のステップで詳しく学ぶ） -->
<input type="text" />
```

`<br />` のようにスラッシュで閉じる書き方と `<br>` の書き方はどちらも有効です。本コンテンツでは `<br />` で統一します。

## まとめ

- ブロック要素は前後に改行が入り、インライン要素は横に並ぶ
- ブロック要素の中にインライン要素を入れる。逆（インラインの中にブロック）は無効
- `p` の中にブロック要素は入れられない
- void要素は終了タグを持たない。`img`・`br`・`hr`・`input` など
